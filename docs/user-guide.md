## 🚀 快速开始

### 1. 系统要求

在开始使用AI-Penetration-Platform之前，请确保您的系统满足以下要求：

- **操作系统**: Linux, macOS, Windows
- **Python**: 3.9+
- **Node.js**: 18+
- **内存**: 4GB+ (推荐8GB)
- **存储**: 10GB+ 可用空间
- **网络**: 稳定的互联网连接

### 2. 安装部署

#### 方法一：Docker 快速部署

```bash
# 克隆项目
git clone https://github.com/your-username/ai-penetration-platform.git
cd ai-penetration-platform

# 启动所有服务
docker-compose up -d

# 访问应用
open http://localhost:3000
```

#### 方法二：手动部署

```bash
# 1. 克隆项目
git clone https://github.com/your-username/ai-penetration-platform.git
cd ai-penetration-platform

# 2. 设置后端
cd backend
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate
pip install -r requirements.txt

# 3. 设置前端
cd ../frontend
npm install

# 4. 配置环境变量
cp .env.example .env
# 编辑 .env 文件配置数据库连接等

# 5. 启动服务
# 后端
cd backend
python -m uvicorn api.main:app --reload --host 0.0.0.0 --port 8000

# 前端 (新终端)
cd ../frontend
npm start
```

## 📋 扫描管理

### 1. 创建扫描

#### 基本信息设置

1. **扫描名称**: 为扫描任务设置一个描述性的名称
2. **扫描类型**: 选择扫描的目标类型
   - **Web应用**: 针对Web应用程序的安全扫描
   - **移动应用**: 针对移动应用程序的安全扫描
   - **网络设备**: 针对网络设备的安全扫描

3. **目标设置**: 
   - **URL**: 输入完整的网站URL (如: `https://example.com`)
   - **IP地址**: 输入目标IP地址 (如: `192.168.1.100`)
   - **域名**: 输入目标域名 (如: `example.com`)

#### 扫描配置

```json
{
  "depth": "deep",
  "include_subdomains": true,
  "max_pages": 100,
  "timeout": 300,
  "user_agent": "Mozilla/5.0 (compatible; AI-Scanner/1.0)",
  "headers": {
    "Accept": "text/html,application/xhtml+xml,application/xml;q=0.9,*/*;q=0.8"
  },
  "cookies": {},
  "auth": {
    "type": "basic",
    "username": "user",
    "password": "pass"
  }
}
```

#### 高级配置选项

- **扫描深度**: 
  - `shallow`: 快速扫描，只检查常见漏洞
  - `medium`: 中等深度扫描，检查主要漏洞类型
  - `deep`: 深度扫描，全面检查所有可能的漏洞

- **并发设置**: 
  - `max_concurrent_scans`: 同时运行的最大扫描数量
  - `request_timeout`: 单个请求的超时时间（秒）

- **扫描范围**:
  - `include_subdomains`: 是否包含子域名
  - `max_pages`: 最大扫描页面数
  - `max_depth`: 最大爬取深度

### 2. 扫描监控

#### 实时进度查看

扫描过程中，您可以：

1. **查看进度**: 实时查看扫描进度百分比
2. **查看日志**: 查看扫描过程中的详细日志
3. **查看统计**: 查看已发现的漏洞统计信息
4. **暂停/继续**: 暂停或继续扫描任务
5. **取消扫描**: 取消正在运行的扫描任务

#### 扫描状态说明

- **等待中 (Pending)**: 扫描任务已创建，等待执行
- **运行中 (Running)**: 扫描任务正在执行中
- **已完成 (Completed)**: 扫描任务已完成，可以查看结果
- **失败 (Failed)**: 扫描任务执行失败，请查看日志
- **已取消 (Cancelled)**: 扫描任务已被取消

## 🎯 风险评估

### 1. 风险评分

AI-Penetration-Platform使用智能算法计算风险分数：

- **0-20分**: 低风险
- **21-40分**: 中风险
- **41-60分**: 高风险
- **61-80分**: 严重风险
- **81-100分**: 极高风险

### 2. 风险因素

#### 技术风险

- **漏洞类型**: SQL注入、XSS、CSRF等
- **严重程度**: 漏洞的严重程度
- **可利用性**: 漏洞的利用难度
- **影响范围**: 漏洞影响的目标数量

#### 业务风险

- **数据敏感性**: 涉及敏感数据的程度
- **业务影响**: 对业务运营的影响
- **合规要求**: 符合法规和标准的要求
- **声誉影响**: 对企业声誉的影响

### 3. 风险建议

#### 修复优先级

1. **立即修复**: 严重和高危漏洞

4. **考虑优化**: 提示级别的问题

#### 修复建议

- **技术建议**: 具体的技术修复方案
- **业务建议**: 业务层面的改进建议

## ❓ 常见问题

### 1. 安装问题

#### Q: Docker启动失败怎么办？
**A**: 请检查Docker是否正确安装，并确保有足够的磁盘空间。可以尝试以下命令：

```bash
# 检查Docker状态
docker --version

# 重启Docker服务
sudo systemctl restart docker

# 检查Docker日志
sudo journalctl -u docker
```

#### Q: Python依赖安装失败怎么办？
**A**: 请确保Python版本为3.9+，并尝试以下方法：

```bash
# 升级pip
python -m pip install --upgrade pip

# 使用国内镜像源
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple/
```

### 2. 扫描问题

#### Q: 扫描任务一直处于等待状态怎么办？
**A**: 检查以下问题：

1. **系统资源**: 确保有足够的系统资源
2. **网络连接**: 确保网络连接正常
3. **扫描队列**: 检查扫描队列是否有积压
4. **配置文件**: 检查配置文件是否正确

#### Q: 扫描任务失败怎么办？
**A**: 请查看扫描日志，常见原因包括：

1. **目标不可达**: 检查目标URL或IP地址是否正确
2. **认证失败**: 检查认证信息是否正确
3. **超时设置**: 调整超时时间设置
4. **权限问题**: 确保有足够的权限访问目标

#### Q: 扫描结果不准确怎么办？
**A**: 可以尝试以下方法：

1. **调整扫描深度**: 增加扫描深度
2. **配置扫描规则**: 自定义扫描规则
3. **更新AI模型**: 更新AI模型到最新版本
4. **手动验证**: 手动验证扫描结果

### 3. 性能问题

#### Q: 系统运行缓慢怎么办？
**A**: 可以尝试以下优化方法：

1. **增加硬件资源**: 增加内存和CPU
2. **优化配置**: 调整系统配置参数
3. **清理缓存**: 清理系统缓存
4. **负载均衡**: 使用负载均衡分散负载

#### Q: 内存使用过高怎么办？
**A**: 可以尝试以下方法：

1. **调整并发数**: 减少并发扫描数量
2. **优化内存使用**: 优化代码内存使用
3. **增加交换空间**: 增加系统交换空间
4. **使用缓存**: 优化缓存策略

### 4. 数据问题

#### Q: 如何备份数据？
**A**: 可以使用以下方法备份数据：

```bash
# 备份数据库
docker exec ai-penetration-platform-postgres pg_dump -U postgres ai_penetration_platform > backup.sql

# 备份上传文件
tar -czf uploads_backup.tar.gz uploads/

# 备份配置文件
cp -r config/ config_backup/
```

#### Q: 如何恢复数据？
**A**: 可以使用以下方法恢复数据：

```bash
# 恢复数据库
docker exec -i ai-penetration-platform-postgres psql -U postgres ai_penetration_platform < backup.sql

# 恢复上传文件
tar -xzf uploads_backup.tar.gz

# 恢复配置文件
cp -r config_backup/* config/
```
